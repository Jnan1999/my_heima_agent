# 扫地机器人智能助理系统

## [项目目标]

设计并实现一个具备长期记忆、多轮对话与精准信息检索能力的智能客服系统，旨在提供超越传统问答的个性化、持续伴随式服务，包括深度故障诊断、个性化使用报告生成与跨会话的用户管理。

---

## [项目介绍]

本项目是一个基于大语言模型的智能助理系统，专为扫地机器人用户设计。系统核心是一个由LangChain与Deep Agents框架构建的ReAct智能体，其独特之处在于深度融合了本地化记忆系统、Skills技能引擎与混合检索技术。前端通过Streamlit构建了交互式Web界面，后端集成通义千问模型与Chroma向量数据库。

### 持久化记忆与个性化服务核心

系统在本地构建了以Markdown文件为核心的双层记忆库。`memory/`目录下存储了结构化的：
- `sessions/`：每次对话的完整会话记录（用户ID_会话ID.md格式）
- `{user_id}/MEMORY.md`：用户长期记忆文件，记录用户偏好、设备状态、历史交互等
- `archives/`：记忆刷盘时的备份归档

每次对话，Agent不仅能访问当前会话历史，更能通过检索这些记忆文件，主动识别用户身份与设备历史，实现"记住用户上次反馈的问题"和"根据用户家庭环境推荐清洁策略"等个性化交互。

### Skills技能引擎

将客服能力模块化为独立的Agent Skills。每个Skill是一个包含自然语言指令（SKILL.md）的文件夹。当用户提出复杂需求时，Agent会调用对应的技能：
- `ask-user-question/`：主动向用户提问获取完整需求
- `memory-brush/`：记忆刷盘技能，当对话产生高价值信息时触发长期记忆更新

### 精准的混合检索与知识管理

系统集成了RAG（检索增强生成）服务：
- `rag/vector_store.py`：基于Chroma的向量存储服务，支持PDF/TXT文档加载、MD5去重、递归分块
- `rag/rag_service.py`：RAG问答服务，实现从检索到生成的完整流程
- 知识库可不断扩充扫地机器人的使用指南、故障处理等专业知识

### 可配置的系统架构

系统的行为逻辑高度可配置：
- `IDENTITY.md`：客服身份定义（名称、形象、说话风格）
- `SOUL.md`：服务人格定义（语气、边界、服务规范）
- `AGENTS.md`：核心操作指令（ReAct流程、工具规则、记忆读写策略）
- `USER_{id}.md`：用户画像（用户ID对应的基本信息、使用特征、偏好）
- 配置文件使用YAML格式管理（config/rag.yml、config/chroma.yml等）

### 前端交互界面

采用IDE风格的三栏式布局：
- 左侧：用户信息输入 + 会话列表管理（新建/删除会话）
- 中间：对话流（居中显示）
- 底部：状态栏（当前日期）

支持：
- 用户ID确认后自动加载历史会话
- 会话自动持久化到本地文件
- 流式输出提供实时打字机效果

### 中间件系统

实现了多个中间件增强Agent能力：
- `monitor_tool`：工具调用监控，记录日志
- `log_before_model`：模型调用前日志
- `report_prompt_switch`：根据上下文自动切换提示词（普通对话/报告生成）

---

## [项目成果]

成功交付了一个具备"记忆力"的智能助理，而非简单的问答机器人。系统能够跨会话识别用户，基于历史交互提供连贯服务。

### 核心技术亮点

- **ReAct思考框架**：遵循"思考→行动→观察→再思考"的推理流程，每步可追溯
- **双层记忆系统**：短期会话记忆 + 长期用户记忆，支持事件触发刷盘
- **Skills技能机制**：复杂任务模块化，通过自然语言指令定义业务流程
- **混合检索**：向量检索 + RAG，支持专业知识库的精准问答

### 技术栈

| 层级 | 技术 |
|------|------|
| Agent框架 | LangChain + Deep Agents |
| 向量数据库 | Chroma |
| 大模型 | 通义千问（Qwen） |
| 前端 | Streamlit |
| 配置管理 | YAML |
| 记忆存储 | Markdown文件 |
