# Ask User Question Skill

## Overview

当用户提出的问题不够清晰、缺少必要的上下文或关键参数时，agent 应该主动向用户提问，而不是盲目猜测或返回不准确的结果。这个 skill 帮助 agent 识别不完整的问题并引导用户补充必要信息。

## When to Use

在以下情况下使用这个 skill：

1. **缺少关键参数**：用户请求中缺少完成操作所必需的信息
2. **意图不明确**：用户的需求有多种可能的理解方式
3. **需要确认**：在执行重要操作前需要用户确认细节
4. **信息模糊**：用户使用模糊的描述（如"那个文件"、"上次的数据"等）

## Instructions

### 步骤 1：分析用户问题

识别用户问题中缺失的关键信息：

- 实体：涉及的对象、文件、人、地点等
- 动作：用户想要做什么
- 条件：时间范围、限制条件、筛选标准等
- 格式：期望的输出格式、结构化程度

### 步骤 2：生成澄清问题

根据缺失的信息生成清晰、具体的问题：

```python
# 分析模式
clarification_questions = []

# 检查缺失的关键信息
if missing_entities:
    question = f"您希望处理哪个/哪些{缺失的实体类型}？"
    clarification_questions.append(question)

if missing_action_context:
    question = f"您希望如何处理这些{实体}？（例如：分析、总结、转换等）"
    clarification_questions.append(question)

if missing_time_range:
    question = "您想关注哪个时间范围的数据？"

if missing_output_format:
    question = "您希望以什么格式输出结果？（如：表格、列表、报告等）"
```

### 步骤 3：向用户提问

使用友好、清晰的方式提问：

- 一次只问 1-2 个最重要的问题
- 提供具体的选项或示例帮助用户理解
- 解释为什么需要这些信息

### 示例对话

**用户**："帮我分析数据"

**Agent（使用此skill）**：
> 为了更好地帮助您分析数据，我需要了解一些细节：
> 
> 1. 您想分析哪些数据？（例如：销售数据、用户行为数据、财务数据等）
> 2. 您希望以什么形式呈现分析结果？（例如：图表、报告、关键指标汇总等）
> 
> 请告诉我更多关于您需求的信息，这样我可以提供更准确的分析。

## 原则

1. **不假设**：不要假设用户的意图，主动确认
2. **提供选项**：尽量提供具体的选项，减少用户的回答难度
3. **解释原因**：说明为什么需要这些信息
4. **逐步澄清**：不要一次问太多问题，逐轮获取信息
5. **记住上下文**：一旦用户提供了信息，在当前对话中记住这些信息
