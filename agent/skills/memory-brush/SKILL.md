# 记忆刷盘技能

## 概述

本技能用于将对话中产生的高价值信息整合到长期记忆（MEMORY.md）中。

## 触发条件

当单次对话中产生以下高价值信息时，必须触发记忆刷盘：

1. **用户明确想要生成使用报告**
2. **用户确认了重要偏好**（如"我喜欢静音模式"、"我家的地面是木地板"）
3. **完成了一个关键故障的排查**（如确定了故障原因并给出了解决方案）
4. **用户提供了新的个人信息**（如家庭成员构成、住房类型变化）

## 输入文件

刷盘时需要读取以下文件作为输入：

1. **现有MEMORY.md** - 长期记忆基线（位于 `memory/{user_id}/MEMORY.md`）
2. **所有会话日志** - 自上次刷盘以来的会话记录（位于 `memory/sessions/{user_id}_{session_id}.md`）
3. **USER.md** - 用户画像（位于 `prompts/users/USER_{user_id}.md`）
4. **AGENTS.md** - 记忆规则（位于 `prompts/AGENTS.md`）

## 处理流程

1. 读取上述所有源文件
2. 识别需要写入长期记忆的信息：
   - 用户偏好（静音模式、清洁时间偏好等）
   - 设备问题及解决方案
   - 用户确认的重要信息
   - 环境特征变化
3. 执行信息整合：
   - 提取新信息
   - 去重（与现有MEMORY.md对比）
   - 纠错（修正错误信息）
   - 总结（用简洁语言描述）
   - 结构化（按类别组织）
4. 生成全新的MEMORY.md内容
5. 备份旧文件到 `memory/archives/`
6. 保存新MEMORY.md

## 输出格式

生成的MEMORY.md应包含以下结构：

```markdown
# 长期记忆

## 用户偏好
- [偏好1]
- [偏好2]

## 设备问题
- [问题1]: [解决方案]
- [问题2]: [解决方案]

## 环境信息
- [环境信息1]
- [环境信息2]

## 历史记录
- [重要事件1]
- [重要事件2]
```

## 重要原则

- 新文件必须**完全替代**旧文件，而非追加
- 旧文件自动备份到 archives 目录
- 仅写入**确认的**信息，未经用户确认的不写入
- 遵循AGENTS.md中的记忆规则判断信息价值
